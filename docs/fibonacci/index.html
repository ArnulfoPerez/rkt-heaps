Fibonacci Heaps

A Fibonacci heap is a data structure for maintaining a collection of elements. In addition to the binomial heap operations, Fibonacci heaps provide two additional operations viz. decrement and delete exist. Although, it should be noted that that trees in Fibonacci heaps are not binomial trees as the implementation cuts subtrees out of them in a controlled way. The rank of a tree is the number of children of the root, and as with binomial heaps we only link two trees if they have the same rank. It is essential here as the rank of root node of a tree is used as the index in the root vector of the heap to store its reference.

procedure

(makeheap val) → heap?

  val : number?
Returns a newly allocated heap with only one element val.

Example:

> (define h (makeheap 3))

procedure

(findmin h) → number?

  h : heap?
Returns a minimum value in the heap h.

Examples:

> (define h (meld (makeheap 1) (makeheap 2)))
> (findmin h)

1

procedure

(insert! h val) → void?

  h : heap?
  val : number?
Updates the heap, h, with a new node having val.

Examples:

> (define h (makeheap 1))
> (insert! h 2)
> (heap-size h)

2

procedure

(deletemin! h) → heap?

  h : heap?
Updates the given heap h by removing the node with the minimum value and changing the reference to the new min node.

Examples:

> (define h (meld (makeheap 1) (makeheap 2)))
> (deletemin! h)
> (findmin h)

2

procedure

(meld h1 h2) → heap?

  h1 : heap?
  h2 : heap?
Returns a newly allocated heap by coupling h1 and h2.

Examples:

> (define h (meld (makeheap 1) (makeheap 2)))
> h

#<heap>

procedure

(decrement! h noderef delta) → void?

  h : heap?
  noderef : node?
  delta : number?
Updates the value of noderef and if the heap condition is violated, then parent of the noderef is checked and if already marked, it is removed. This happens recursively until the root of the tree is reached or a parent which is not marked.

Examples:

> (define h (meld (makeheap 1) (makeheap 2)))
> (decrement! h (heap-minref h) 2)
> (findmin h)

-1

procedure

(delete! h noderef) → void?

  h : heap?
  noderef : node?
Updates the heap h by deleting the noderef and updating its parent and children. Here also if the parent is marked, then it is also removed from the tree and added as a root. Happens until the root of the tree is reached or a parent is not marked.

Examples:

> (define h (meld (makeheap 1) (makeheap 2)))
> (delete! h (heap-minref h))
> (findmin h)

2

> (heap-size h)

1

procedure

(heap-minref h) → node?

  h : heap?
Returns the reference of the node which has the minimum value in h.

procedure

(heap-roots h) → vector?

  h : heap?
Returns a vector with references to all roots in h. The references are indexed in the vector based on the rank of the root.

procedure

(heap-size h) → exect-nonnegative-integer?

  h : heap?
Returns the size of the heap h.

procedure

(node-val n) → number?

  n : node?
Returns the value stored in the node n.

procedure

(node-children n) → vector?

  n : node?
Returns a vector of all children of node n. If n does not have any children then a empty vector will be returned.

procedure

(node-parent n) → node?

  n : node?
Returns the parent node of n if there exists one or #f.
